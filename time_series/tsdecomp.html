def to_data_range(date_str):
    return '20' + date_str[-2:] + '-' + date_data[date_str[:3]][0] + '-' + date_data[date_str[:3]][0]

def correl(per_max=36, series=[]):

    maxCorr = 0.0
    period = np.nan
    for i in range(1, per_max):
        corr = series.autocorr(lag=i)
        if corr > maxCorr:
            maxCorr = corr
            period = i
    return {
        "Period": period,
        "Max_Correlation": maxCorr
    }

Conclusions - Private Construction¶

There are definite limitations to the moving average approach to this dataset, specifically because the trend in the dataset is heavily influenced by periods that are more recent. This means that the larger window which was attempted made the performance of the model worse. I did try an exponential moving average, and the alpha level which worked best was around a=0.75 which would indicate the trend in the data is volatile and recency has a higher weight.

Aside from trend, there is a clear, precise annual trend to this data, with peaks in the summer months in the US (May, June, July, August) and troughs in the winter (November, December, January, Febuary) with ramp ups and ramp downs in the months adjacent to these. This is likely explained by weather and condition favorability of the Summer months with that of the winter.

Conclusion - Public Construction¶

Like with private construction, there are limits to the application of the moving average technique for this data set. While I believe the model is satisfactory given the data. I believe there are potentially better methods which could be used, possibly exponential smoothing.

Likewise with Private Construction, there is a strong annual cycle present in the data. This cyclical pattern is the same as the one that I addressed in my earlier conclusion section.
